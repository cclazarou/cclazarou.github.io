Introduction

The Korg Electribe 2 is a popular music production station/sampler/sequencer. It comes with a large bank of "radio ready" preset sounds, and seems to lost a little cred amongst underground electronic and noise musicians because of its ability to provide pre-baked compositions. However, it has fairly granular controls over sound for a cheap-ish workstation, and I've really enjoyed using it to become more comfortable with sequencing in composition.

This blog is a compilation of my expansions on the Electribe user manuals, as well as an exploration the workflow that works for me.

What it can do

IFX vs. MFX

The Electribe allows each user to add effects in two ways: IFX (insert effects) or MFX (master effects)

IFX

You can add an effect to a Part by doing the following:

  1. Press the Trigger button
  2. On the Trigger Pad, press the Part that you'd like to apply the IFX to
  3. Press the IFX on button
  4. Use the Insert FX pot to select the effect you prefer
  5. Use the Edit pot (under the Insert FX pot) to select the dry/wet ratio. It's listed as a number as if it's a level, but dry/wet is implied.

An Insert Effect is added to the singular Part before its audio hits the main audio outputs. You can assign one insert effect per Part.

MFX

You can add an effect to a Part or multiple Parts by sending it to the MFX bus:

  1. Press the Trigger button
  2. On the Trigger Pad, select the Part you'd like to apply the Master Effect to
  3. Press the Master FX Button, as well as the Master Hold button, underneath the display.[1]
  4. Press the MFX Send button under the Amp/EG section of pots to tell the Electribe you want to send the Part to the MFX bus
  5. Select the effect that will be applied to all Parts being sent to the MFX bus by selecting the effect in the display menu (Menu button -> MFX Type, use Value knob to select).

Keep in mind that you can only apply one MFX to every Part being sent to the MFX bus, and that the list of Insert Effects and list of Master Effects are different (ex: there is no reverb IFX, unfortunately)

Sampling

Although there are sampling tools built into the Electribe, I find their editing tools to be too crude (specifically if I want to shorten a sample, which more often than not causes a distinct clipping sound). Instead, I opt to edit the sample in a digital audio workstation, export it as a WAV to an Electribe-formatted SD card, and import it to the Electribe

  • If you're not using an SD card already, navigate to the formatting tool inside of the display menu to format a new SD card.
  • Turn off the Electribe and remove the SD card
  • In your digital audio workstation on your computer, make all your edits/add effects and bounce a WAV file of the exact length you want to use in the Electribe
  • Connect the SD card to your computer, and place a copy of the WAV file in the Samples folder. This is a good time to copy the contents of the entire SD card to a safe place.
  • Insert the SD card in the Electribe
  • The Electribe automatically loads all samples from the SD card into its internal drive. This is done by overwriting the current samples folder stored internally. If you switch SD cards without transferring sample data between the cards, keep in mind that you will be erasing the current contents of the internal Sample folder with the contents of your new Sample folder (which may delete internal Samples). I'm not sure if this applies to preset sounds as well, but it's all the more reason to back up the contents of your SD card with regularity.
[1]The manual indicates a completely different function for MFX Hold, and does not specify this needs to be done to send a Part to the MFX bus. However, I've found that I can't apply the MFX to a Part without pressing this button. I'll update as I get more information.

What's wrong with the other Electribe resources

I mean the following in an Actionable, Specific, and Kind way, of course. This is in reference to Practical Music Production.

Good

  • grid provides engaging alternative to navbar
  • slider provides users a quick guided summary of the site
  • social media icons are in SVG format

Bad

  • use of label tag in association with something other than an input tag (I may be wrong about this, W3C explicitly states that it's used in conjunction with an input tag, but MDN describes it as a bit more flexible
  • the whitespace is excessive, and makes for a less smooth reading experience
  • the alt attribute is used with the a tag, which is common but not valid according to W3C
  • empty classes
  • relying on break tag for spacing
  • unstyled bullets look unpolished
